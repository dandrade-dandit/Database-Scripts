Library Cache Misses indicate that the Shared Pool is not big
enough to hold the shared SQL area for all concurrently open cursors.

prevents ORACLE from deallocating a shared SQL area while an
application

processes have had to wait for space in the redo buffer.  If this
condition exists over time, increase the size of LOG_BUFFER in the
init.ora file in increments of 5% until the value nears 0.
** NOTE: increasing the LOG_BUFFER value will increase total SGA size.

LATCH CONTENTION:

GETS - # of successful willing-to-wait requests for a latch
MISSES - # of times an initial willing-to-wait request was unsuccessful
IMMEDIATE_GETS - # of successful immediate requests for each latch
IMMEDIATE_MISSES = # of unsuccessful immediate requests for each latch
SLEEPS - # of times a process waited and requests a latch after an
initial willing-to-wait request

If the latch requested with a willing-to-wait request is not
available, the requesting process waits a short time and requests
again.
If the latch requested with an immediate request is not available,
the requesting process does not wait, but continues processing

Parse Ratio usually falls between 1.15 and 1.45.  If it is higher, then
it is usually a sign of poorly written Pro* programs or unoptimized
SQL*Forms applications.

Recursive Call Ratio will usually be between

7.0 - 10.0 for tuned production systems
10.0 - 14.5 for tuned development systems

Buffer Hit Ratio is dependent upon RDBMS size, SGA size and
the types of applications being processed.  This shows the %-age
of logical reads from the SGA as opposed to total reads - the
figure should be as high as possible.  The hit ratio can be raised
by increasing DB_BUFFERS, which increases SGA size.  By turning on
the "Virtual Buffer Manager" (db_block_lru_statistics = TRUE and
db_block_lru_extended_statistics = TRUE in the init.ora parameters),
you can determine how many extra hits you would get from memory as
opposed to physical I/O from disk.  **NOTE:  Turning these on will
impact performance.  One shift of statistics gathering should be enough
to get the required information.
